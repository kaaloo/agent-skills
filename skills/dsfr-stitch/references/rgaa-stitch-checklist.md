# RGAA preflight for Stitch artifacts

This checklist is a preflight only. Final code needs the full RGAA skill audit.

## Visual and content checks

- Page has one clear `h1` and a coherent heading hierarchy.
- French copy is institutional, concrete, and user-task oriented.
- No invented statistics, fake organization names, or placeholder content remains.
- Color choices are DSFR tokens or explicitly justified exceptions.
- Text/background contrast appears compliant before formal testing.
- Focus states are visible in prototype or specified for implementation.

## Structural checks

- Header, breadcrumb, main region, and footer roles are planned.
- Forms have visible labels, help text, required-state indication, and error messages.
- Buttons and links are distinguishable by role and label.
- Tables have captions and scoped headers when data is tabular.
- Images have a planned alternative: informative alt, decorative empty alt, or text alternative.
- Modals, menus, tabs, and accordions have keyboard behavior defined for implementation.

## Handoff checks

- Each Stitch screen has a target DSFR or LaSuite component mapping.
- Any custom component need is named and justified.
- Responsive behavior is specified for mobile and desktop.
- Final implementation plan includes real-browser verification and RGAA audit.
